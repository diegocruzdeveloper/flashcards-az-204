Flashcards de Estudo para AZ-204 (150 Cartões)

Develop Azure compute solutions (30)

FRENTE

O que é o Azure Container Registry (ACR)?

VERSO

Um serviço de registro privado e gerenciado para imagens de contêiner Docker e OCI, permitindo construir, armazenar e gerenciar imagens.

FRENTE

O que é o Azure Container Instances (ACI)?

VERSO

A maneira mais rápida e simples de executar um contêiner no Azure, sem gerenciar servidores ou orquestradores. Ideal para tarefas simples e automação.

FRENTE

O que é o Azure Container Apps (ACA)?

VERSO

Uma plataforma serverless sobre Kubernetes, para rodar microsserviços em contêineres. Inclui escalonamento, Ingress e Dapr.

FRENTE

O que são Deployment Slots?

VERSO

Ambientes "live" para validar alterações em staging antes do "swap" com produção, garantindo implantações com zero downtime.

FRENTE

O que é um Plano do App Service?

VERSO

Define os recursos de computação (região, SKU, VMs) para seus Web Apps. Vários apps podem compartilhar o mesmo plano.

FRENTE

O que são Durable Functions?

VERSO

Uma extensão do Azure Functions para escrever fluxos de trabalho stateful (que guardam estado) em um ambiente serverless.

FRENTE

Diferença entre Scale-up e Scale-out?

VERSO

Scale-up (vertical): Aumenta os recursos da VM (CPU, memória).
Scale-out (horizontal): Aumenta o número de instâncias da VM.

FRENTE

O que são ACR Tasks?

VERSO

Um conjunto de recursos dentro do Azure Container Registry para construir, testar e aplicar patches em imagens de contêiner na nuvem, de forma automatizada.

FRENTE

Quais os 3 tipos de gatilhos (triggers) de Azure Functions?

VERSO

1. Timer (agendado), 2. Data operations (ex: novo blob, mensagem na fila), 3. Webhooks (requisição HTTP).

FRENTE

O que é a configuração WEBSITE_RUN_FROM_PACKAGE?

VERSO

Uma configuração de App Service/Function App que monta um arquivo ZIP (de uma URL) como o diretório `wwwroot` somente leitura. Melhora a performance de deploy e evita conflitos de lock de arquivos.

FRENTE

O que são os Planos de Hospedagem de Azure Functions?

VERSO

Consumption (Paga pelo uso, escala auto), Premium (Instâncias pré-aquecidas, sem cold start, VNet), e Dedicated/App Service Plan (Recursos fixos).

FRENTE

Para que serve o Dapr (Distributed Application Runtime) no ACA?

VERSO

Abstrai desafios comuns de microsserviços, como chamadas serviço-a-serviço, pub/sub, gerenciamento de estado e segredos, através de APIs sidecar.

FRENTE

O que é a VNet Integration no App Service?

VERSO

Permite que seu aplicativo acesse recursos em uma Rede Virtual (VNet) do Azure, como VMs, bancos de dados ou outros serviços que não estão expostos publicamente.

FRENTE

O que são Revisões (Revisions) em Azure Container Apps?

VERSO

Cada vez que você atualiza a imagem do contêiner ou a configuração, uma nova revisão imutável é criada. O ACA permite dividir o tráfego entre diferentes revisões (ex: para testes A/B).

FRENTE

Como configurar diagnóstico e logging no App Service?

VERSO

Através das "Configurações de diagnóstico", enviando logs de aplicação, servidor web e métricas para o Log Analytics, Storage Account ou Event Hubs.

FRENTE

O que são Custom Handlers em Azure Functions?

VERSO

Permitem usar qualquer linguagem ou runtime que suporte HTTP (Go, Rust, etc.) para criar uma Azure Function, bastando expor um web server local.

FRENTE

Para que serve o arquivo host.json?

VERSO

Contém opções de configuração globais que afetam todas as funções em uma Function App, como logging, timeouts e configurações de extensões (bindings).

FRENTE

O que é a configuração CORS no App Service?

VERSO

Cross-Origin Resource Sharing. Permite que um script em uma página web de uma origem (domínio) acesse recursos (APIs) em outra origem.

FRENTE

O que é o App Service Authentication (Easy Auth)?

VERSO

Um recurso que integra autenticação (Microsoft Entra ID, Google, etc.) ao seu App Service ou Function App com configuração mínima, sem alterar o código.

FRENTE

Quais os níveis de autorização de um HTTP Trigger?

VERSO

Anonymous (sem chave), Function (requer chave de função), e Admin (requer a chave mestre).

FRENTE

Como funciona o TimerTrigger?

VERSO

Executa uma função em um agendamento definido por uma expressão CRON. Ex: `0 */5 * * * *` executa a cada 5 minutos.

FRENTE

O que é a Geo-replicação no ACR?

VERSO

Um recurso do tier Premium que replica as imagens do seu registro para outras regiões do Azure, permitindo deploys mais rápidos e resilientes.

FRENTE

Como os slots de implantação herdam configurações?

VERSO

Por padrão, todas as configurações são trocadas (swapped) com o slot de produção, a menos que sejam marcadas como "Deployment slot setting".

FRENTE

O que é Idempotência em Azure Functions?

VERSO

É o princípio de que executar a mesma operação múltiplas vezes produz o mesmo resultado que executá-la uma única vez. Importante para sistemas de mensagens com entrega "at-least-once".

FRENTE

O que são "service connections" no App Service?

VERSO

Refere-se a como seu aplicativo se conecta a outros serviços (banco de dados, storage, etc.), geralmente gerenciado por Connection Strings ou referências a Key Vault.

FRENTE

O que é o padrão Singleton em Azure Functions?

VERSO

Garante que apenas uma instância de uma função seja executada por vez (em todas as VMs). Útil para evitar condições de corrida em operações não-idempotentes.

FRENTE

Como configurar TLS mútuo (mTLS) no App Service?

VERSO

Configurando os "Client certificates", onde o App Service exige que o cliente apresente um certificado para se autenticar, além do servidor apresentar o seu.

FRENTE

O que é um Health Check no App Service?

VERSO

Um endpoint que o App Service pode "pingar" para determinar se uma instância está saudável. Se não estiver, ela é removida do load balancer e reiniciada.

FRENTE

O que é um "cold start" em Functions de Consumo?

VERSO

O tempo de latência inicial para executar uma função que esteve inativa. Acontece porque a infraestrutura precisa ser alocada e o app iniciado.

FRENTE

Como funciona o deploy de código a partir do GitHub Actions para o App Service?

VERSO

Usando um workflow YAML no repositório GitHub que define os passos para build, teste e deploy, utilizando um "publish profile" para autenticar no Azure.

Develop for Azure storage (30)

FRENTE

Quais as 5 APIs do Cosmos DB?

VERSO

NoSQL (Core/SQL), MongoDB, Cassandra, Gremlin (grafo), e Table.

FRENTE

O que é o Change Feed no Cosmos DB?

VERSO

Um log persistente de todas as alterações em um contêiner, usado para criar lógicas reativas, como acionar uma Azure Function.

FRENTE

Qual a importância da Partition Key no Cosmos DB?

VERSO

Distribui dados e requisições de forma uniforme entre as partições. Uma boa chave evita "hot partitions" e garante escalabilidade.

FRENTE

O que são Request Units (RU/s) no Cosmos DB?

VERSO

A moeda de performance que abstrai CPU, IOPS e memória. Você provisiona a taxa de transferência (throughput) em RU/s.

FRENTE

Quais as Camadas de Acesso (Access Tiers) do Blob Storage?

VERSO

Hot (frequente), Cool (infrequente), Cold (raro), e Archive (offline). O custo de armazenamento diminui, mas o de acesso aumenta.

FRENTE

O que é o Gerenciamento do Ciclo de Vida no Blob Storage?

VERSO

Regras baseadas em políticas para automatizar a transição de blobs entre camadas ou para excluí-los, otimizando custos.

FRENTE

O que é o Controle de Concorrência Otimista no Cosmos DB?

VERSO

Usa a propriedade de sistema `_etag` (entity tag) para prevenir "lost updates". A operação falha se o ETag no servidor for diferente do ETag enviado pelo cliente.

FRENTE

O que é TTL (Time-to-Live) no Cosmos DB?

VERSO

Uma configuração que permite ao Cosmos DB excluir itens automaticamente de um contêiner após um certo período de tempo (em segundos).

FRENTE

Diferença entre Block, Page e Append Blobs?

VERSO

Block: Para upload de grandes arquivos (vídeos, imagens). Page: Para discos de VM (operações de leitura/escrita aleatórias). Append: Para logs (operações de anexar).

FRENTE

O que é a Exclusão reversível (Soft Delete) para blobs?

VERSO

Protege contra exclusão acidental, mantendo os dados excluídos por um período de retenção configurável, permitindo sua recuperação.

FRENTE

O que são Índices Compostos no Cosmos DB?

VERSO

Índices definidos em múltiplas propriedades para otimizar queries que filtram ou ordenam por mais de um campo, reduzindo o consumo de RUs.

FRENTE

O que são os modos de throughput do Cosmos DB?

VERSO

Provisioned (manual ou autoscale): você define as RU/s. Serverless: paga por RU consumida, ideal para cargas de trabalho esporádicas.

FRENTE

Para que serve o Versioning de blobs?

VERSO

Cria automaticamente versões anteriores de um blob quando ele é modificado ou excluído, permitindo restaurar estados passados.

FRENTE

O que são Propriedades vs Metadados de um blob?

VERSO

Propriedades: Atributos do sistema (Content-Type, etc.).
Metadados: Pares chave-valor definidos pelo usuário para informações personalizadas.

FRENTE

O que é Hospedagem de site estático no Storage?

VERSO

Um recurso que permite servir conteúdo estático (HTML, CSS, JS) diretamente de um contêiner especial (`$web`) em uma conta de armazenamento.

FRENTE

O que é a Change Feed Processor Library?

VERSO

Uma biblioteca que simplifica a leitura do Change Feed do Cosmos DB, gerenciando o estado (checkpoints) e a distribuição do trabalho entre múltiplos consumidores.

FRENTE

O que são as Políticas de Imutabilidade para blobs?

VERSO

Armazenam dados em um estado WORM (Write-Once, Read-Many), tornando-os não-apagáveis e não-modificáveis por um intervalo de tempo (Time-based) ou indefinidamente (Legal hold).

FRENTE

O que são Blob Index Tags?

VERSO

Atributos de índice de dados (chave-valor) que permitem categorizar e encontrar blobs específicos usando um serviço de consulta nativo, sem listar todos os contêineres.

FRENTE

O que é um Lease de blob?

VERSO

Um bloqueio (lock) de escrita e exclusão em um blob. Garante que apenas um cliente possa modificar ou apagar o blob por vez.

FRENTE

O que é o Azure Data Lake Storage Gen2?

VERSO

É um conjunto de recursos para análise de big data, construído sobre o Azure Blob Storage, que adiciona um namespace hierárquico e semântica de sistema de arquivos.

FRENTE

Para que serve o AzCopy?

VERSO

Uma ferramenta de linha de comando para copiar dados de e para o Azure Blob Storage, File Storage e Table Storage com performance otimizada.

FRENTE

`UpsertItemAsync` vs `ReplaceItemAsync` no SDK do Cosmos?

VERSO

Upsert: Cria o item se não existir, ou o substitui se existir.
Replace: Substitui um item existente; falha se o item não existir.

FRENTE

O que são Stored Procedures no Cosmos DB (API NoSQL)?

VERSO

Lógicas escritas em JavaScript que são executadas atomicamente dentro de uma partição lógica, permitindo transações em múltiplos documentos.

FRENTE

Como o SDK do Cosmos DB lida com retries?

VERSO

O SDK possui uma política de retry embutida que tenta novamente as operações em caso de erros transitórios, como throttling (HTTP 429).

FRENTE

O que é um Contêiner de Lease no Change Feed?

VERSO

É um contêiner auxiliar usado pela Change Feed Processor Library para armazenar o estado (checkpoints) do processamento das partições do contêiner monitorado.

FRENTE

O que é um Blob Snapshot?

VERSO

Uma versão somente leitura de um blob tirada em um ponto específico no tempo. Útil para backups.

FRENTE

Para que serve o consistency level Session?

VERSO

Garante que, dentro de uma única sessão de cliente, as leituras sejam consistentes com as escritas ("read-your-own-writes"). É o nível de consistência padrão.

FRENTE

O que é Autoscale Throughput no Cosmos DB?

VERSO

Gerencia automaticamente as RU/s provisionadas, escalando para cima ou para baixo com base no uso, dentro de um intervalo máximo definido por você.

FRENTE

Como você implementa uma transação no Cosmos DB?

VERSO

Usando Stored Procedures (para múltiplos documentos na mesma partição lógica) ou o recurso de Transactional Batch no SDK (.NET/Java).

FRENTE

Posso consultar dados na camada Archive do Blob Storage?

VERSO

Não diretamente. Você precisa primeiro "reidratar" o blob para a camada Hot ou Cool, o que pode levar várias horas.

Implement Azure security (30)

FRENTE

O que é a Plataforma de Identidade da Microsoft?

VERSO

Serviço de identidade na nuvem (Microsoft Entra ID) que permite aos apps logar usuários e obter tokens para chamar APIs protegidas.

FRENTE

O que é uma Shared Access Signature (SAS)?

VERSO

Uma URI que concede direitos de acesso restritos e delegados a recursos do Storage, com permissões e tempo de validade, sem expor a chave da conta.

FRENTE

O que é a API do Microsoft Graph?

VERSO

Uma API RESTful com um ponto de extremidade unificado (`graph.microsoft.com`) para acessar dados de serviços da nuvem da Microsoft (Entra ID, Office 365).

FRENTE

Qual a função do Azure Key Vault?

VERSO

Um serviço para armazenar e gerenciar segredos de forma segura: chaves de API, senhas, connection strings, certificados e chaves de criptografia.

FRENTE

O que são Managed Identities?

VERSO

Uma identidade gerenciada no Microsoft Entra ID para um recurso do Azure se autenticar em outros serviços, eliminando credenciais no código.

FRENTE

App Configuration vs Key Vault?

VERSO

App Configuration: para configurações e feature flags. Key Vault: para segredos (senhas, chaves). Podem ser usados juntos.

FRENTE

O que é a MSAL (Microsoft Authentication Library)?

VERSO

Uma biblioteca de cliente que simplifica a obtenção de tokens da Plataforma de Identidade da Microsoft para autenticar usuários e acessar APIs.

FRENTE

O que são App Roles?

VERSO

Permissões personalizadas para uma aplicação (ex: "Admin", "Reader"). Usuários ou outras aplicações podem ser atribuídos a esses papéis para controle de acesso.

FRENTE

Diferença: System-assigned vs User-assigned Managed Identity?

VERSO

System: Ligada ao ciclo de vida de um recurso (criada e deletada com ele).
User: Recurso independente que pode ser atribuído a múltiplos recursos.

FRENTE

O que é um Service Principal (Entidade de Serviço)?

VERSO

É a instância de uma aplicação (App Registration) em um tenant do Entra ID. É a identidade local da aplicação, à qual permissões são concedidas.

FRENTE

O que é um token JWT?

VERSO

JSON Web Token. Um padrão aberto para criar tokens de acesso que contêm "claims" (informações) como um objeto JSON. Usado para autenticação e autorização.

FRENTE

Diferença: Key Vault Access Policies vs RBAC?

VERSO

Access Policies: Modelo antigo, permissões por cofre para um principal (usuário/app).
RBAC: Modelo mais novo e granular, permite atribuir roles (ex: Key Vault Secrets User) em escopos diferentes.

FRENTE

O que são scopes na plataforma de identidade?

VERSO

Definem as permissões específicas que uma aplicação cliente está solicitando para acessar uma API em nome de um usuário (ex: `User.Read`, `Mail.Send`).

FRENTE

O que é a DefaultAzureCredential?

VERSO

Uma classe nos SDKs do Azure que tenta se autenticar usando vários mecanismos em sequência (variáveis de ambiente, Managed Identity, Visual Studio, etc.). Simplifica a autenticação no desenvolvimento e produção.

FRENTE

Quais os 3 tipos de SAS?

VERSO

1. User Delegation SAS (mais segura, usa credenciais Entra ID), 2. Service SAS (acesso a um recurso específico), 3. Account SAS (acesso a múltiplos serviços da conta).

FRENTE

O que são Stored Access Policies para SAS?

VERSO

Políticas de acesso definidas no lado do servidor (em um contêiner, fila, etc.) que controlam as permissões e validade de um SAS, permitindo revogá-lo facilmente.

FRENTE

Diferença: Permissões Delegadas vs de Aplicação no Graph?

VERSO

Delegadas: O app age em nome de um usuário logado. De Aplicação: O app tem permissão própria, sem usuário presente (para daemons, serviços de backend).

FRENTE

O que é a validação de token em uma API?

VERSO

O processo de verificar a assinatura, o emissor (issuer), o público (audience) e a validade (expiration) de um token de acesso para garantir que ele é autêntico e válido.

FRENTE

O que é a proteção contra expurgo (purge protection) no Key Vault?

VERSO

Quando habilitada junto com a exclusão reversível (soft delete), impede que segredos/chaves excluídos sejam permanentemente apagados (purgados) antes do fim do período de retenção.

FRENTE

O que é o Princípio do Menor Privilégio?

VERSO

Conceder a uma identidade (usuário, app) apenas as permissões mínimas necessárias para realizar suas tarefas, e nada mais.

FRENTE

Como usar um certificado do Key Vault em um App Service?

VERSO

Importando o certificado para o App Service a partir do Key Vault e depois o vinculando a um domínio personalizado nas configurações de TLS/SSL.

FRENTE

O que é o fluxo On-Behalf-Of (OBO)?

VERSO

Um fluxo de autorização onde uma API de camada intermediária usa a identidade de um usuário para obter um token de acesso para chamar outra API (downstream).

FRENTE

O que são as Políticas de Acesso Condicional?

VERSO

Regras no Microsoft Entra ID que avaliam sinais (usuário, localização, dispositivo) para impor controles de acesso, como exigir MFA ou bloquear o acesso.

FRENTE

O que é o App Registration (Registro de App)?

VERSO

A definição global da sua aplicação no Microsoft Entra ID. Descreve o que o app pode fazer, quem pode usá-lo e quais APIs ele pode acessar.

FRENTE

Como implementar RBAC no código?

VERSO

Verificando as "claims" de papéis (`roles`) presentes no token de acesso do usuário para autorizar ou negar o acesso a certas funcionalidades.

FRENTE

O que é um cliente confidencial vs público?

VERSO

Confidencial: Apps que rodam em servidores e podem guardar segredos (APIs, web apps).
Público: Apps que não podem guardar segredos (SPAs, apps mobile).

FRENTE

O que é o Microsoft Defender for Cloud?

VERSO

Uma ferramenta de gerenciamento de postura de segurança e proteção contra ameaças que ajuda a fortalecer a segurança dos seus recursos na nuvem.

FRENTE

Para que servem os grupos de segurança de rede (NSG)?

VERSO

Atuam como um firewall para filtrar o tráfego de rede de e para recursos do Azure em uma VNet, com base em regras de segurança.

FRENTE

O que é o Azure Firewall?

VERSO

Um serviço de segurança de rede gerenciado e stateful que protege os recursos da sua VNet, filtrando tráfego com base em regras de aplicação, rede e inteligência de ameaças.

FRENTE

O que é uma referência do Key Vault no App Configuration?

VERSO

É um valor no App Configuration que aponta para um segredo no Key Vault. O app obtém o segredo do cofre de forma transparente, usando a identidade do App Configuration.

Monitor and troubleshoot Azure solutions (30)

FRENTE

O que é o Application Insights?

VERSO

Um serviço de APM (Application Performance Management) para monitorar aplicações, detectar anomalias, diagnosticar problemas e entender o uso.

FRENTE

O que é instrumentar uma aplicação?

VERSO

Significa adicionar o SDK do Application Insights ao código (ou habilitar auto-instrumentação) para coletar e enviar dados de telemetria.

FRENTE

O que são Testes de Disponibilidade?

VERSO

Testes automatizados que enviam requisições web para sua aplicação de vários pontos do mundo para verificar disponibilidade e responsividade.

FRENTE

Para que serve a Kusto Query Language (KQL)?

VERSO

A linguagem usada para fazer consultas avançadas sobre os dados de logs e telemetria no Application Insights e Azure Monitor Logs.

FRENTE

Diferença: Métricas vs Logs?

VERSO

Métricas: Valores numéricos coletados em intervalos regulares (contadores, gauges).
Logs: Eventos com carimbo de data/hora e propriedades detalhadas.

FRENTE

O que é a Rastreabilidade Distribuída (Distributed Tracing)?

VERSO

A capacidade de rastrear uma única requisição através de múltiplos serviços (microsserviços), correlacionando as chamadas para entender o fluxo completo e identificar gargalos.

FRENTE

O que é o Application Map?

VERSO

Uma visualização dos componentes da sua aplicação e suas dependências, mostrando o fluxo de chamadas e indicadores de performance (taxa de sucesso, duração).

FRENTE

O que são Action Groups?

VERSO

Uma coleção de ações (email, SMS, webhook, etc.) que podem ser acionadas por um alerta do Azure Monitor. São reutilizáveis em múltiplos alertas.

FRENTE

O que é Sampling (Amostragem) no App Insights?

VERSO

Uma técnica para reduzir o volume (e custo) de telemetria, enviando apenas uma porcentagem dos dados, mantendo a correlação entre os itens.

FRENTE

O que são Eventos Personalizados (Custom Events)?

VERSO

Telemetria que você envia do seu código para rastrear ações específicas do usuário ou eventos de negócio (ex: `TrackEvent("ItemAdicionadoAoCarrinho")`).

FRENTE

O que é o Log Analytics Workspace?

VERSO

O ambiente do Azure Monitor para armazenar, gerenciar e analisar dados de logs de vários recursos do Azure e de ambientes on-premises.

FRENTE

O que o Profiler do App Insights faz?

VERSO

Captura automaticamente traços detalhados do desempenho de requisições lentas em produção, ajudando a identificar o "hot path" (caminho quente) no código.

FRENTE

O que é o Snapshot Debugger?

VERSO

Coleta automaticamente um snapshot de depuração (variáveis locais, call stack) quando uma exceção ocorre em produção, sem parar a execução da aplicação.

FRENTE

Como você rastreia dependências no App Insights?

VERSO

A auto-instrumentação captura automaticamente chamadas para dependências externas, como bancos de dados SQL, APIs HTTP e Storage.

FRENTE

O que são Workbooks no Azure Monitor?

VERSO

Relatórios visuais interativos que combinam texto, consultas de logs, métricas e parâmetros em um único documento, para análise e troubleshooting.

FRENTE

O que é a Smart Detection?

VERSO

Um recurso que usa machine learning para analisar proativamente a telemetria e alertar sobre possíveis problemas de performance ou falhas.

FRENTE

Como a interface ILogger se integra com o App Insights?

VERSO

Em apps .NET, logs escritos via ILogger (`_logger.LogInformation`, etc.) são automaticamente capturados pelo SDK do App Insights e enviados como telemetria de `trace`.

FRENTE

Qual o propósito de `TrackException()`?

VERSO

Para enviar manualmente informações sobre exceções para o Application Insights, especialmente útil para exceções tratadas (em blocos `catch`) que você ainda quer monitorar.

FRENTE

O que é um Telemetry Processor?

VERSO

Um componente do pipeline de coleta que permite modificar ou filtrar a telemetria no cliente antes que ela seja enviada para o Azure (ex: remover PII).

FRENTE

O que o Activity Log do Azure registra?

VERSO

Eventos no nível da subscrição, como a criação, modificação ou exclusão de recursos do Azure (o "quem, o quê e quando" das operações no ARM).

FRENTE

O que é Live Metrics?

VERSO

Um stream de métricas de performance (requisições, CPU, etc.) com latência de ~1 segundo, útil para monitorar um novo deploy ou diagnosticar um problema em tempo real.

FRENTE

O que a tabela `requests` contém em KQL?

VERSO

Registra detalhes sobre cada requisição recebida pela sua aplicação, como URL, duração, código de resultado, e ID de correlação.

FRENTE

O que é um alerta de métrica?

VERSO

Um alerta que é acionado quando o valor de uma métrica cruza um limiar definido (ex: CPU > 90% por 5 minutos).

FRENTE

O que é um alerta de log?

VERSO

Um alerta que é acionado com base no resultado de uma consulta KQL executada em intervalos regulares (ex: alertar se o número de exceções > 10 na última hora).

FRENTE

Como você pode analisar falhas no App Insights?

VERSO

Através do painel "Failures", que agrupa exceções e requisições falhas, permitindo analisar detalhes, contar ocorrências e ver a transação de ponta a ponta.

FRENTE

Como você mede a performance no lado do cliente (browser)?

VERSO

Instrumentando suas páginas web com o SDK JavaScript do Application Insights, que coleta telemetria de visualizações de página, chamadas AJAX e exceções no browser.

FRENTE

Qual o papel do ID de Correlação?

VERSO

Conecta toda a telemetria (requisições, dependências, traces) gerada por uma única operação, permitindo a rastreabilidade distribuída.

FRENTE

O que é o Azure Monitor Agent (AMA)?

VERSO

O novo agente para coletar logs e métricas de VMs, substituindo agentes legados. É configurado via "Data Collection Rules".

FRENTE

Onde os dados do App Insights são armazenados?

VERSO

Em um Log Analytics Workspace (modo baseado em workspace) ou em um armazenamento clássico do App Insights (que está sendo preterido).

FRENTE

O que é Resource Health?

VERSO

Um serviço do Azure que informa sobre a saúde dos seus recursos individuais, diagnosticando problemas na plataforma Azure que possam estar te afetando.

Connect to and consume Azure services (30)

FRENTE

O que é o Azure API Management (APIM)?

VERSO

Um serviço para publicar, proteger e monitorar APIs. Atua como um gateway na frente dos seus serviços de back-end.

FRENTE

O que são Políticas (Policies) no APIM?

VERSO

Regras que são executadas no request/response, permitindo modificar o comportamento da API sem alterar o código (ex: validar JWT, rate limiting).

FRENTE

Event Grid vs Event Hubs?

VERSO

Event Grid: Roteamento de eventos discretos (pub/sub reativo).
Event Hubs: Streaming de big data e telemetria contínua.

FRENTE

O que são Consumer Groups no Event Hubs?

VERSO

Representam uma visão do stream de eventos. Cada app consumidor usa seu próprio grupo para ler o stream de forma independente.

FRENTE

Service Bus vs Queue Storage?

VERSO

Queue Storage: Fila simples e barata.
Service Bus: Robusto, com FIFO, transações, tópicos/assinaturas (pub/sub), sessões, DLQ.

FRENTE

O que é uma Dead-Letter Queue (DLQ)?

VERSO

Uma fila secundária que armazena mensagens que não puderam ser processadas ou entregues, permitindo inspecioná-las e reprocessá-las.

FRENTE

O que são Produtos e Assinaturas no APIM?

VERSO

Produtos: Agrupam uma ou mais APIs. Assinaturas: Concedem acesso a um produto através de uma chave de API, permitindo controle de acesso e cotas.

FRENTE

Diferença: Service Bus Topics vs Queues?

VERSO

Queues: Entregam uma mensagem para um único consumidor.
Topics: Modelo publish/subscribe, entregam uma mensagem para múltiplos assinantes (subscriptions).

FRENTE

O que são Partições no Event Hubs?

VERSO

Subdivisões do stream de eventos. Permitem o consumo paralelo do stream, aumentando a taxa de transferência (throughput).

FRENTE

O que é a filtragem de eventos no Event Grid?

VERSO

A capacidade de uma assinatura de evento receber apenas um subconjunto dos eventos publicados em um tópico, com base no tipo de evento ou em propriedades no corpo do evento.

FRENTE

Modos de recebimento no Service Bus: Peek-Lock vs Receive-and-Delete?

VERSO

Peek-Lock (padrão): Bloqueia a mensagem, o cliente a processa e a completa (deleta).
Receive-and-Delete: A mensagem é deletada assim que é recebida.

FRENTE

O que é o Azure Cache for Redis?

VERSO

Um serviço de cache distribuído e gerenciado, baseado no Redis. Usado para melhorar a performance e escalabilidade de aplicações, armazenando dados frequentemente acessados na memória.

FRENTE

O que são os escopos de política no APIM?

VERSO

Políticas podem ser aplicadas em 4 níveis (do mais amplo para o mais específico): Global, Produto, API e Operação.

FRENTE

O que é o recurso Capture do Event Hubs?

VERSO

Captura e armazena automaticamente os eventos de um Event Hub em um Blob Storage ou Data Lake Storage, em formato Avro.

FRENTE

O que são Sessões no Service Bus?

VERSO

Garantem o processamento ordenado (FIFO) e exclusivo de um grupo de mensagens relacionadas, identificadas por um `SessionId`.

FRENTE

O que é o Schema Registry no Event Hubs?

VERSO

Um repositório centralizado para esquemas de eventos (ex: Avro), permitindo que produtores e consumidores validem e serializem/desserializem dados de forma consistente.

FRENTE

O que é o Developer Portal no APIM?

VERSO

Um site gerado automaticamente e customizável onde desenvolvedores podem descobrir suas APIs, ler a documentação, obter chaves e testar as operações.

FRENTE

O que é o timeout de visibilidade no Queue Storage?

VERSO

O tempo que uma mensagem fica invisível na fila após ser lida. Se o processamento não for concluído e a mensagem deletada nesse tempo, ela reaparece na fila.

FRENTE

O que são Tópicos do Sistema no Event Grid?

VERSO

Tópicos embutidos para serviços do Azure (Blob Storage, Service Bus, etc.) que publicam eventos automaticamente, facilitando a integração.

FRENTE

O que é a detecção de duplicados no Service Bus?

VERSO

Um recurso que descarta mensagens duplicadas enviadas para uma fila ou tópico dentro de uma janela de tempo configurável, com base no `MessageId`.

FRENTE

Como lidar com mensagens venenosas (poison messages) no Queue Storage?

VERSO

Verificando a propriedade `DequeueCount`. Se exceder um limite, a lógica do seu app deve mover a mensagem para uma fila separada de "venenosas" para análise.

FRENTE

O que é uma Named Value (Propriedade) no APIM?

VERSO

Uma variável global (chave-valor) que pode ser usada em políticas do APIM. Pode ser um valor literal ou um segredo importado do Key Vault.

FRENTE

O que são Filtros em uma assinatura do Service Bus Topic?

VERSO

Regras que permitem que uma assinatura receba apenas um subconjunto das mensagens publicadas no tópico, com base em suas propriedades (ex: `color = 'blue'`).

FRENTE

Qual o caso de uso para a política `set-backend-service` no APIM?

VERSO

Para rotear dinamicamente uma requisição para um serviço de back-end diferente do que está configurado na API, com base em condições na política.

FRENTE

O que é a biblioteca `EventProcessorClient`?

VERSO

Uma classe no SDK do Event Hubs que simplifica o consumo de eventos, gerenciando a distribuição de partições entre consumidores e o checkpointing.

FRENTE

O que é a funcionalidade de autoforwarding no Service Bus?

VERSO

Permite encadear filas ou assinaturas, fazendo com que mensagens de uma entidade sejam automaticamente movidas para outra, sem código.

FRENTE

O que é o dead-lettering no Event Grid?

VERSO

Uma configuração que envia eventos que não puderam ser entregues a um endpoint (após retries) para uma conta de armazenamento para análise posterior.

FRENTE

Quando usar o Azure SignalR Service?

VERSO

Para adicionar comunicação web em tempo real (push de conteúdo do servidor para o cliente) a aplicações, como chats, dashboards e notificações ao vivo.

FRENTE

Qual a diferença entre API Gateway e API Management?

VERSO

O Gateway é o componente que roteia as requisições. O Management é a solução completa que inclui o gateway, um portal do desenvolvedor e um plano de controle para gerenciamento.

FRENTE

O que é o schema CloudEvents?

VERSO

Uma especificação da CNCF para descrever dados de eventos de forma comum, visando a interoperabilidade entre diferentes serviços e plataformas.
